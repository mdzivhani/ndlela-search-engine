# Frontend Standards

## Structure

- Use React with TypeScript or Express where required.
- Keep components reusable and modular.

Folder convention:

src/
  components/
  pages/
  services/
  hooks/
  state/

## Naming

- PascalCase for components.
- camelCase for functions and instances.
- Use clear names aligned with functionality.

## UX and Accessibility

- Follow WCAG accessibility principles.
- Keep spacing consistent.
- Reuse UI elements where possible.
