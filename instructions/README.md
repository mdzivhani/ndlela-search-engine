## Purpose

These guidelines help teams:

- Maintain consistent code quality across projects
- Onboard new developers efficiently
- Make code review processes more effective
- Ensure maintainability and scalability
- Establish clear architectural boundaries

## Usage

Pick and adapt guidelines that fit your project's needs. Not all recommendations will apply to every project or technology stack.
